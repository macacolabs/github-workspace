GitHub 원격 저장소

로컬 Git 저장소를 GitHub에 연결하고, push·pull·clone·fetch를 사용해 원격과 동기화합니다.

🎯 이 챕터를 마치면
  • GitHub에 원격 저장소를 생성하고 연결할 수 있다
  • git push로 로컬 코드를 GitHub에 올릴 수 있다
  • git pull로 원격 변경사항을 받을 수 있다
  • git clone으로 기존 저장소를 복사할 수 있다
  • push/pull/fetch/clone의 차이를 이해한다

☁️ 1. 로컬 vs 원격 저장소

로컬 ↔ GitHub 원격 저장소 관계
💻
로컬 저장소
내 컴퓨터 .git
인터넷 없어도 OK
git push →
git pull / fetch ←
🌐
GitHub (origin)
클라우드 저장소
팀원과 공유

🔗 2. 원격 저장소 연결

1
GitHub에 새 Repository 생성
GitHub → New repository → 이름 입력 → Create repository
2
로컬과 연결
Git Bash
# 원격 저장소 연결 (origin이라는 이름으로)
$ git remote add origin https://github.com/사용자명/저장소명.git

# 연결 확인
$ git remote -v
origin  https://github.com/사용자명/저장소명.git (fetch)
origin  https://github.com/사용자명/저장소명.git (push)
💡

origin은 원격 저장소의 별명(nickname)입니다. 관례적으로 기본 원격 저장소를 origin이라 부릅니다.

🚀 3. push / pull / fetch / clone

git push — 업로드

Git Bash
# 처음 push (upstream 설정)
$ git push -u origin main
Branch 'main' set up to track remote branch 'main' from 'origin'.
To https://github.com/사용자명/저장소명.git
   * [new branch]      main -> main

# 이후부터는 간단하게
$ git push

git pull — 다운로드 + 병합

Git Bash
# 원격 변경사항 가져와서 자동 merge
$ git pull origin main
# upstream 설정 후에는
$ git pull
pull vs fetch 차이
git pull
= fetch + merge
원격 다운로드 후 자동 병합
빠르지만 예상치 못한 충돌 가능
git fetch
다운로드만 (병합 X)
원격 변경사항을 미리 확인
신중한 방식, 충돌 전에 검토

git clone — 저장소 복사

Git Bash
# 원격 저장소를 통째로 복사 (원격 연결 자동 설정됨)
$ git clone https://github.com/사용자명/저장소명.git
Cloning into 'repo-name'...
remote: Enumerating objects: 10, done.
...
Resolving deltas: done.

# 폴더 이름 지정해서 clone
$ git clone https://github.com/사용자명/저장소명.git my-project
명령어 용도 전제
git clone 저장소 최초 복사 원격 저장소가 이미 존재
git push 로컬 → 원격 업로드 원격 연결 완료
git pull 원격 → 로컬 (자동 병합) 원격 연결 완료
git fetch 원격 → 로컬 (수동 병합) 원격 연결 완료

🎯 4. 실습 — 첫 push 완성

1
GitHub에서 새 Repository 생성
README 없이 생성 (로컬 내용을 올릴 것이므로)
2
원격 연결 및 push
Git Bash
$ git remote add origin https://github.com/사용자명/my-first-git.git
$ git branch -M main
$ git push -u origin main
3
GitHub에서 확인
브라우저에서 https://github.com/사용자명/my-first-git에 접속, 파일이 업로드됐는지 확인!
✅ 이해도 체크
origin이 무엇인지 설명할 수 있다
git push -u origin main과 이후의 git push 차이를 안다
pull vs fetch의 차이를 설명할 수 있다
GitHub에 내 코드가 성공적으로 올라가 있다
📌 이 챕터 명령어
git remote add origin URL
원격 저장소 연결
git remote -v
연결된 원격 저장소 확인
git push -u origin main
최초 push (upstream 설정)
git pull
원격 변경사항 받기 + 자동 merge
git fetch
원격 변경사항 받기 (merge 없이)
git clone URL
원격 저장소 복사
