환경 세팅

Git Bash 설치부터 GitHub 계정 연동까지, 실습을 시작하기 위한 모든 준비를 합니다.

🎯 이 챕터를 마치면
  • Git Bash를 설치하고 기본 설정을 완료할 수 있다
  • GitHub 계정을 생성하고 Personal Access Token을 발급할 수 있다
  • git config로 사용자 정보를 설정할 수 있다
  • Fork(GUI 도구)를 설치하고 이해할 수 있다

⬇️ 1. Git 설치

1
Git 공식 사이트 접속
https://git-scm.com 에 접속해서 운영체제에 맞는 설치 파일을 다운로드합니다.
🪟
Windows
Git for Windows
Git Bash 포함
🍎
macOS
Homebrew로 설치
brew install git
🐧
Linux
패키지 매니저
apt install git
2
설치 확인
설치 완료 후 Git Bash를 열고 아래 명령어로 설치를 확인합니다.
Git Bash
$ git --version
git version 2.43.0.windows.1
✅

버전 번호가 출력되면 설치 성공! 버전이 2.x대이면 모두 괜찮습니다.

⚙️ 2. Git 사용자 정보 설정

Git은 커밋을 기록할 때 누가 만들었는지를 반드시 함께 저장합니다. 아래 명령어로 이름과 이메일을 설정해야 합니다.

git config의 3가지 범위
💻
--system
시스템 전체
이 컴퓨터의 모든 사용자에게 적용 • /etc/gitconfig
👤
--global
내 계정 전체 ← 우리가 사용할 옵션
현재 사용자의 모든 Git 프로젝트에 적용 • ~/.gitconfig
📁
--local
프로젝트별
해당 저장소에만 적용 • .git/config

사용자 이름과 이메일 설정

Git Bash
# 이름 설정 (GitHub에 표시될 이름)
$ git config --global user.name "홍길동"

# 이메일 설정 (GitHub 계정 이메일과 동일하게)
$ git config --global user.email "hong@example.com"

# 기본 브랜치 이름을 main으로 설정 (권장)
$ git config --global init.defaultBranch main

# 설정 확인
$ git config --list
user.name=홍길동
user.email=hong@example.com
init.defaultbranch=main
...
⚠️

GitHub 계정 이메일과 동일한 이메일을 입력하세요. 다르면 GitHub에서 커밋을 내 계정과 연결하지 못합니다.

설정한 정보가 커밋에 반영되는 방식
commit a3f82bc...
Author: 홍길동 <hong@example.com> ← 여기에 들어감!
Date: Thu Feb 27 17:00:00 2025
첫 번째 커밋 메시지

🐙 3. GitHub 계정 생성 & 토큰 발급

1
GitHub 회원가입
https://github.com에 접속 → Sign Up 클릭 → 이메일, 비밀번호, 닉네임 입력 후 가입
2
Personal Access Token(PAT) 발급

GitHub는 비밀번호 대신 토큰을 인증에 사용합니다.

토큰 발급 경로
프로필 이미지 클릭
→
Settings
→
Developer settings
→
Personal access tokens
→
Generate new token (classic)
⚠️

토큰은 생성 직후 한 번만 보여줍니다. 반드시 메모장에 저장해두세요!

💡

권한(Scope)은 repo만 체크하면 됩니다. 만료 기간은 90일 이상 설정 권장.

🖥️ 4. Fork — GUI 도구 소개

이 강의는 Git Bash(CLI)로 실습합니다. 하지만 내부 동작을 시각적으로 확인할 수 있는 GUI 도구 Fork도 함께 소개합니다.

우리가 배울 방식
⌨️
Git Bash (CLI)
✅ 실습 주 도구
✅ 어디서든 사용 가능
✅ 실전에서 사용
✅ Git의 모든 기능
+
🖱️
Fork (GUI)
📌 이해 보조 도구
📌 브랜치 시각화
📌 커밋 그래프 확인
📌 진행 상황 눈으로 확인

Fork 설치

https://git-fork.com 에서 다운로드 → 설치 시 이름/이메일 입력 (git config와 동일하게)

💡

Fork 없이도 모든 실습 가능합니다. Fork는 Git의 동작을 눈으로 확인하는 용도로만 사용합니다.

🔐 5. Windows 인증 정보 관리

GitHub에 처음 push/pull할 때 Windows에서 자격증명을 묻습니다. 토큰을 저장하는 방법입니다.

Git Bash
# 자격증명 캐시 저장 (15분)
$ git config --global credential.helper cache

# 자격증명 영구 저장 (Windows)
$ git config --global credential.helper wincred
⚠️ 토큰 초기화 방법 (Windows)

제어판 → 사용자 계정 → 자격 증명 관리자 → Windows 자격 증명 → github.com 관련 항목 제거

✅ 설치 완료 체크
git --version 명령어가 정상 출력된다
이름과 이메일을 git config --global로 설정했다
GitHub 계정을 생성했다
Personal Access Token을 발급하고 안전한 곳에 저장했다
📌 이 챕터 명령어
git --version
Git 버전 확인
git config --global user.name "이름"
전역 사용자 이름 설정
git config --global user.email "이메일"
전역 이메일 설정
git config --list
현재 설정 목록 확인
